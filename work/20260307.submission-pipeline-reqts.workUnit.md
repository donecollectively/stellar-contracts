Four components in the submission pipeline lack architecture documentation and requirements files:

| Component | File | Current State |
|-----------|------|---------------|
| `TxBatcher` | `src/networkClients/TxBatcher.ts` | No arch doc, no reqts file. Manages batch lifecycle, holds `BatchSubmitController` instances. Being extended with tracker registry responsibility. |
| `TxSubmitMgr` | `src/networkClients/TxSubmitMgr.ts` | No reqts file. `BatchSubmitController` has inline prose requirements but no standalone file. Manages per-submitter submit/confirm state machine. |
| `TxSubmissionTracker` | `src/networkClients/TxSubmissionTracker.ts` | No reqts file. Orchestrates multiple `TxSubmitMgr` instances for a single transaction. |
| `BatchSubmitController` | `src/networkClients/BatchSubmitController.ts` | Has inline `reqts()` method with prose requirements but no reqts file. Manages signing and submission for a batch of transactions. |
