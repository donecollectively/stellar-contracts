## Findings

### Finding F1 — RESOLVED
**Severity**: Objection
**Location**: In `CachedUtxoIndex.registerPendingTx` (wraps write sequence in withWriteLock) at `CachedUtxoIndex.ts:1448`, and `CachedUtxoIndex.confirmPendingTx` (wraps output verification) at `CachedUtxoIndex.ts:1592`

**Observation**: Write lock tables lists `["pendingTxs", "utxos", "records", "txs"]` and `["pendingTxs", "utxos", "records"]` are missing `"scripts"`. Inside the locked callback, `indexUtxoFromOutput()` calls `cacheRefScriptIfPresent()` → `store.saveScript()`, writing to the `scripts` Dexie table. Since `scripts` is not in the Dexie transaction's table scope, this throws `TransactionInactiveError` for any output with a reference script (common for delegate UTxOs).

**Resolution**: Added `"scripts"` to both tables lists. Build verified clean.

### Finding F2 — RESOLVED
**Severity**: Concern
**Location**: In `CachedUtxoIndex.confirmPendingTx` (status update after output verification) at `CachedUtxoIndex.ts:~1670-1677`

**Observation**: The `savePendingTx(pendingEntry)` call that writes the confirmed status, slot, hash, and confirmState was outside any write lock. REQT/phj1ne600k requires "wrap output verification and status update." Another tab could read stale pending status between output indexing and status write.

**Resolution**: Wrapped the `savePendingTx` call in a second `withWriteLock("confirmPendingTx-status", ["pendingTxs"])`. Kept the Blockfrost fallback and confirmState calculation outside the lock to avoid holding it during network I/O.

### Finding F3 — RESOLVED
**Severity**: Concern
**Location**: In `CachedUtxoIndex.markInitialSyncDone` and `CachedUtxoIndex.waitForInitialSyncDone` at `CachedUtxoIndex.ts:632-680`

**Observation**: Both methods cast `this.store as any` to directly access Dexie's `store.metadata.put()` and `store.metadata.get()`, bypassing the `UtxoStoreGeneric` interface. This breaks the store abstraction — non-Dexie stores won't have `.metadata`, and the `if (store.metadata?.put)` guard silently no-ops, meaning non-owner tabs would spin for 2 minutes then fall through. Contradicts REQT/n11m1k9wvg (Non-Transactional Store Compatibility).

**Resolution**: Added `getMetadata(key)` and `setMetadata(key, value)` to `UtxoStoreGeneric` interface, implemented in `DexieUtxoStore`, and replaced the `as any` casts in both methods.
