## Problem / Context

### Unit mismatch in CachedUtxoIndex.now

The CardanoClient interface has a `now` property. All implementations return milliseconds since epoch:

- **BlockfrostV0Client**: `Date.now()` ‚Äî wall-clock milliseconds
- **Emulator**: `SECOND * this.currentSlot` ‚Äî simulated milliseconds (SECOND = 1000)
- **CachedUtxoIndex**: `this.lastSlot` ‚Äî **raw slot number** (~142M vs ~1.7T for milliseconds)

`TxSubmitMgr.currentSlot` (line 1133) calls `timeToSlot(this.network.now)`. The `timeToSlot` helper on `NetworkParamsHelper` expects milliseconds: `refTipSlot + Math.round((time - refTipTime) / (1000 * secondsPerSlot))`. When fed a raw slot number, the result is wildly incorrect, making `isTxExpired()` unreliable for expiry detection and validity diagnostics.

In production (browser), `CachedUtxoIndex` IS used as `setup.network` ‚Äî set in `CapoDappProvider`.

### StellarTxnContext.txnTime uses stale tip time

`StellarTxnContext.txnTime` (line 722) defaults to `latestTipTime` from `networkParams` ‚Äî the timestamp of whatever block was the tip when `parameters` was last fetched. This could be minutes old. The transaction's `validFromTime` is set from this value, meaning the validity window starts at a stale past time rather than the current moment. The name `latestTipTime` is misleading ‚Äî it's actually "tip time at last params fetch."

`txnTime` should use `this.setup.network.now` (current wall-clock time). A configurable backdating buffer (default 30s) should be subtracted to ensure the validity start isn't ahead of the chain's consensus view, since wall clock can run slightly ahead of the chain tip.

## Plan / Guidance

### Change 1: Fix CachedUtxoIndex.now

Change `get now()` to return `Date.now()`. This matches `BlockfrostV0Client.now` exactly.

No `lastBlockTime` conversion needed for the `.now` property ‚Äî wall clock is the correct value for the CardanoClient contract.

### Change 2: Rename lastSlot ‚Üí lastBlockSlot, add lastBlockTime

Rename `lastSlot` to `lastBlockSlot` for clarity ‚Äî this is the slot of the last block seen, used internally for slot-to-slot comparisons (confirmation depth, deadline checks). These internal uses are correct and unchanged.

Add `lastBlockTime` (Blockfrost Unix seconds from block's `time` field) alongside `lastBlockSlot` at all update sites. This is a clarity improvement and makes the block timestamp available for future use.

### Change 3: Fix StellarTxnContext.txnTime to use current network time with backdating

Change the `txnTime` getter default from:
```typescript
new Date(makeNetworkParamsHelper(this.networkParams).latestTipTime)
```
to:
```typescript
new Date(this.setup.network.now - backdateBufferMs)
```

The backdating buffer (default 30 seconds, configurable) ensures the validity start isn't ahead of the chain's consensus view. The buffer should be configurable on `SetupInfo` or a similar shared config surface ‚Äî exact placement TBD during implementation.

### Change 4: Remove TxSubmitMgr.currentSlot, fix isTxExpired and checkTxValidityDetails

Remove the `currentSlot` getter entirely ‚Äî it derived a slot from `timeToSlot(network.now)`, which produced garbage when network was CachedUtxoIndex and is conceptually wrong even with the `.now` fix (wall-clock-derived slots can be slightly ahead of chain tip).

- **`isTxExpired`**: Change to compare `lastValidSlot` against the most recent actual observed block slot from the network. Duck-type via `"lastBlockSlot" in this.network` ‚Äî CachedUtxoIndex exposes this as a public property after the rename; fall back to `timeToSlot(network.now)` for other `CardanoClient` implementations (BlockfrostV0Client, Emulator) that don't track block slots. This is the conservative, ground-truth approach ‚Äî a tx is expired only when the chain has definitively moved past its validity window.
- **`checkTxValidityDetails`**: Change to derive the current slot via `timeToSlot(network.now)` for diagnostic display. This is appropriate for human-readable diagnostics ("valid for +N seconds") where approximate wall-clock comparison is useful.

### Change 5: Update test assertions

Update `02e-CachedUtxoIndex.test.ts` line 244 to reflect that `now` returns `Date.now()` instead of `lastSlot`.
