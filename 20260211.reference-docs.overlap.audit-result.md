## Findings

#### Finding: f1a — On-chain vs Off-chain Roles duplication
**Status**: resolved
**Location**: In "On-chain vs. Off-chain Roles" section of essential-stellar-offchain.md:11-16 and essential-cardano.md:3-6

**Observation**: The on-chain validates / off-chain constructs explanation appears in both essential-cardano.md (canonical home) and essential-stellar-offchain.md (near-duplicate with agent summary). Maintenance risk: divergent wording over time.

**Resolution**: Trimmed offchain version to a one-liner referencing cardano.md, keeping agent-specific framing.

#### Finding: f2a — CapoLifecycleActivity internal duplication in lifecycle doc
**Status**: resolved
**Location**: In "CapoLifecycleActivities in mint/spend delegates" (lines 40-47) and "CapoLifecycleActivity responsibility detail" (lines 49-55) of essential-capo-lifecycle.md

**Observation**: Lines 40-47 and 49-55 are near-identical descriptions of the same responsibility split. The onchain doc (essential-stellar-onchain.md lines 113-124) has a third version at the same abstraction level but serves a different audience (on-chain script authors). Not true cross-doc duplication — different scopes — but the internal duplication in the lifecycle doc is unnecessary.

**Resolution**: Removed the duplicate "responsibility detail" section (lines 49-55) from the lifecycle doc. Added cross-references: lifecycle doc points to onchain doc for script-level enforcement and to bootstrapping doc for off-chain interfaces. Onchain doc header clarified as "(on-chain enforcement)" and points back to lifecycle doc for general responsibility model and to bootstrapping for off-chain interfaces.

#### Finding: f3a — UTxO/value helper triple-listing within onchain doc
**Status**: resolved
**Location**: In "Helper modules" (lines 46-65), "On-chain utility functions" (lines 128-134), and "Helper data types and functions" (lines 136-152) of essential-stellar-onchain.md

**Observation**: The same helpers (mkTv, tvCharter, mustFindInputRedeemer, fromCip68Wrapper, outputAndDatum, REQT/logGroup) appeared three times in the same file at different abstraction levels. Parameter signatures, struct fields, and `getOutputForInput` existed only in the middle listing.

**Resolution**: Consolidated into the "Helper modules" section with full narrative descriptions, source file paths, parameter signatures, and struct field names. Absorbed unique items (time helpers, returnsValueToScript, getOutputForInput) from removed sections. No information lost.

#### Finding: f4a — Component/delegate descriptions across three files
**Status**: resolved
**Location**: In "Components" of essential-stellar-dapp-architecture.md:30-41, "Core classes" of essential-stellar-offchain.md:18-23, and "Capo validator" of essential-stellar-onchain.md:15-24

**Observation**: Core component descriptions appear in all three files. Each serves a distinct audience (architecture, off-chain API, on-chain enforcement) but the relationship between the three perspectives was not made explicit, risking confusion about whether they describe the same or different things.

**Resolution**: Added clarifying sentences in each doc: architecture doc notes off-chain and on-chain perspectives exist elsewhere; offchain doc notes the on-chain classes have similar naming but are distinct, providing on-chain enforcement; onchain doc notes the similarly-named off-chain classes provide transaction-building and lifecycle access.

#### Finding: f5a — Bootstrap/lifecycle flows across four files
**Status**: resolved
**Location**: In "Software Development Lifecycle" of essential-stellar-dapp-architecture.md:71-75, "Kickstart/Overview" of essential-offchain-bootstrapping.md:5-15, "Bootstrap / lifecycle flow pointers" of essential-capo-lifecycle.md:20-24, and "Charter lifecycle" of essential-stellar-onchain.md:166-171

**Observation**: Bootstrap/lifecycle steps described in four files. Each serves its audience (architecture overview, actionable checklist, responsibility model, on-chain enforcement). The onchain doc's charter lifecycle section referenced essential-capo-lifecycle.md but not the bootstrapping doc for off-chain interfaces.

**Resolution**: Added cross-link from onchain doc's charter lifecycle section to essential-offchain-bootstrapping.md for the off-chain transaction-building side. No content removed — the brief summaries in architecture and lifecycle docs are appropriate as pointers.

#### Finding: f6a — Responsibility snapshot near-duplicate between architecture + lifecycle
**Status**: resolved
**Location**: In "Responsibilities" of essential-stellar-dapp-architecture.md:90-95 and "Responsibility snapshot" of essential-capo-lifecycle.md:15-18

**Observation**: The mint/spend/Capo responsibility breakdown is nearly word-for-word identical in both files. The lifecycle doc is the canonical home; the architecture doc already referenced it at line 95.

**Resolution**: Replaced the architecture doc's "Responsibilities" subsection with a one-liner deferring to essential-capo-lifecycle.md for the full breakdown. The lifecycle doc retains the complete responsibility snapshot.

#### Finding: f7a — Data/UTxO model overlap between architecture + onchain
**Status**: resolved
**Location**: In "Data/UTxO model" of essential-stellar-dapp-architecture.md:57-71 and "Core on-chain types" of essential-stellar-onchain.md:76-90

**Observation**: Both files describe the data model — architecture doc conceptually (what lives where), onchain doc as Helios types (CapoDatum variants, CIP-68 structure). Complementary perspectives but neither acknowledged the other.

**Resolution**: Added reciprocal cross-links: architecture doc points to onchain for Helios type definitions; onchain doc points to architecture for the conceptual data model.
